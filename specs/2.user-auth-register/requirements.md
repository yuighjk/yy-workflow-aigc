# user-auth-register — 需求规格

## 概述

提供 `/register` 注册页面：用户名 + 邮箱 + 密码 + 确认密码输入、前端基础校验、loading 反馈，校验通过后在页面内展示"注册成功，请登录"，并提供返回登录的入口。复用 feature 1 建立的认证面板 shell 与视觉规范。

## 项目信息

- 项目名: yy-workflow-aigc
- 架构类型: monorepo（Turborepo + pnpm workspaces，全栈 TypeScript）
- 本 feature 仅涉及**前端层**（`apps/web`），不涉及后端 / 数据库。

## 用户故事

- 作为新用户，我希望打开注册页面填写账号信息，以便创建账户。
- 作为用户，我希望表单错误（含两次密码不一致）能被清楚提示，以便修正。
- 作为用户，我希望注册成功后得到明确提示，并被引导去登录。
- 作为用户，我希望能从注册页一键切换回登录页。

## 功能需求

1. [F-001] `/register` 路由与注册页面：含产品名、注册标题，复用认证面板 shell。
2. [F-002] 用户名、邮箱、密码、确认密码 4 个输入框。
3. [F-003] 基础校验：用户名不能为空、邮箱不能为空、邮箱格式合法、密码不能为空、密码长度 ≥ 6、确认密码必须与密码一致。
4. [F-004] 注册按钮：点击后进入 loading 状态（按钮文案"注册中..."、禁用），用 300–500ms 模拟请求延迟。
5. [F-005] 校验通过后在页面内展示"注册成功，请登录"提示，并提供返回登录入口。
6. [F-006] 跳转登录入口：链接跳转到 `/login`。
7. [F-007] 表单错误提示展示在对应输入框下方（含"两次输入的密码不一致"）。
8. [F-008] 桌面端与移动端均正常展示，元素无重叠或溢出。

## 非功能需求

- 性能: 页面轻量；模拟延迟 300–500ms，不阻塞输入。
- 安全: 纯前端 mock，不采集/发送真实凭据；遵循 `.claude/rules/security.md`。
- 兼容性: 响应式，面板宽度 360–420px，最小高度 100vh。
- 可访问性: `label` 关联 input、语义化 `<form>`/`<button>`、错误文案可读。

## 验收标准

- [ ] [AC-001] 用户可以打开 `/register` 注册页面。
- [ ] [AC-002] 注册页包含用户名、邮箱、密码、确认密码、注册按钮和跳转登录链接。
- [ ] [AC-003] 表单为空时点击注册，展示对应错误提示。
- [ ] [AC-004] 邮箱格式错误时，展示邮箱格式错误提示。
- [ ] [AC-005] 密码少于 6 位时，展示密码长度错误提示。
- [ ] [AC-006] 两次密码不一致时，展示"两次输入的密码不一致"提示。
- [ ] [AC-007] 点击注册后按钮进入"注册中..."的 loading 态（300–500ms）。
- [ ] [AC-008] 校验通过后页面展示"注册成功，请登录"，并可点击入口返回 `/login`。
- [ ] [AC-009] 桌面端与移动端展示正常，无重叠/溢出。

## 依赖

- 前端框架与库（已在 `apps/web` 安装）：`@tanstack/react-router`、`@tanstack/react-form`、`zod`、`@yy-workflow-aigc/ui`、Tailwind CSS v4。
- 跨 feature：**依赖 feature 1（user-auth-login）** 的认证面板 shell（`auth/auth-panel.tsx`）与视觉 tokens；`2.T-001 依赖 1.T-001`。`/login` 路由由 feature 1 提供。
