# user-auth-login — 需求规格

## 概述

提供 `/login` 登录页面：邮箱 + 密码输入、前端基础校验、loading 反馈，校验通过后在页面内展示"登录成功"（不接后端、不跳转），并提供跳转注册的入口。

## 项目信息

- 项目名: yy-workflow-aigc
- 架构类型: monorepo（Turborepo + pnpm workspaces，全栈 TypeScript）
- 本 feature 仅涉及**前端层**（`apps/web`），不涉及后端 / 数据库。

## 用户故事

- 作为已有用户，我希望打开登录页面输入邮箱和密码，以便进入系统。
- 作为用户，我希望表单错误能被清楚提示在对应输入框下方，以便知道如何修正。
- 作为用户，我希望登录成功后看到明确的页内反馈，以便确认操作完成。
- 作为用户，我希望能从登录页一键切换到注册页。

## 功能需求

1. [F-001] `/login` 路由与登录页面：含产品名、登录标题，居中认证面板布局。
2. [F-002] 邮箱输入框 + 密码输入框。
3. [F-003] 基础校验：邮箱不能为空、邮箱格式合法、密码不能为空、密码长度 ≥ 6。
4. [F-004] 登录按钮：点击后进入 loading 状态（按钮文案"登录中..."、禁用），用 300–500ms 模拟请求延迟。
5. [F-005] 校验通过后在页面内展示绿色"登录成功"提示条；**不跳转页面**，登录按钮恢复可点击。
6. [F-006] 跳转注册入口：链接跳转到 `/register`（feature 2 提供该路由）。
7. [F-007] 表单错误提示展示在对应输入框下方（示例文案："请输入邮箱""请输入正确的邮箱格式""密码至少 6 位"）。
8. [F-008] 桌面端与移动端均正常展示，元素无重叠或溢出。

## 非功能需求

- 性能: 页面首屏轻量；模拟延迟 300–500ms，不阻塞输入。
- 安全: 纯前端 mock，不采集/发送真实凭据；遵循 `.claude/rules/security.md`（不硬编码密钥、不引入服务端密钥到前端包）。
- 兼容性: 响应式，主流现代浏览器；面板宽度 360–420px，最小高度 100vh。
- 可访问性: `label` 与 input 关联、语义化 `<form>`/`<button>`、错误文案可读（遵循 Ultracite 无障碍约定）。

## 验收标准

- [ ] [AC-001] 用户可以打开 `/login` 登录页面。
- [ ] [AC-002] 登录页包含邮箱、密码、登录按钮和跳转注册链接。
- [ ] [AC-003] 表单为空时点击登录，展示对应错误提示。
- [ ] [AC-004] 邮箱格式错误时，展示邮箱格式错误提示。
- [ ] [AC-005] 密码少于 6 位时，展示密码长度错误提示。
- [ ] [AC-006] 点击登录后按钮进入"登录中..."的 loading 态（300–500ms）。
- [ ] [AC-007] 校验通过后页面展示绿色"登录成功"提示条，且不跳转。
- [ ] [AC-008] 点击注册链接可跳转到 `/register`。
- [ ] [AC-009] 桌面端与移动端展示正常，无重叠/溢出。

## 依赖

- 前端框架与库（已在 `apps/web` 安装）：`@tanstack/react-router`、`@tanstack/react-form`、`zod`、`@yy-workflow-aigc/ui`（Card/Button/Input/Label）、Tailwind CSS v4。
- 跨 feature：`/register` 路由由 feature 2（user-auth-register）提供；开发期若 feature 2 未就绪，注册链接可先指向占位路径。
